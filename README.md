# 链接标记器 (Link Marker) 插件使用教程

## 一、插件安装步骤

### 1. 开启开发者模式
1. 打开 Chrome 浏览器
2. 点击右上角的三个点 → 更多工具 → 扩展程序
3. 在扩展程序页面右上角，开启「开发者模式」开关

### 2. 加载已解压的扩展程序
1. 点击「加载已解压的扩展程序」按钮
2. 在弹出的文件选择窗口中，选择 `link_mark` 文件夹（整个插件项目的根目录）
3. 点击「选择文件夹」按钮
4. 此时插件应该已经成功加载到浏览器中，会显示在扩展程序列表中

### 3. 固定插件图标（可选）
1. 点击浏览器右上角的拼图图标
2. 在弹出的扩展程序列表中，找到「链接标记器」
3. 点击插件名称右侧的图钉图标，将其固定到浏览器工具栏

## 二、插件功能使用说明

### 1. 弹窗页面 (Popup)
点击浏览器工具栏中的插件图标，会弹出一个小窗口，包含以下功能：

- **全局启用/禁用**：控制插件的整体开关状态
- **一键清除所有标记**：清除所有网页的链接标记效果和本地存储的标记数据
- **选项设置**：打开详细的选项配置页面
- **历史记录**：打开标记历史记录页面

### 2. 选项设置页面 (Options)
在弹窗页面点击「选项设置」按钮，进入详细的配置页面，包含以下设置项：

#### 2.1 样式自定义
- **前缀图标样式**：选择标记链接前的图标样式（圆点、对勾、五角星、数字）
- **图标大小**：调整图标大小（10-20px）
- **文本样式**：设置链接文字的样式（加粗、倾斜、删除线）
- **启用背景色**：控制是否显示链接的背景高亮
- **背景色**：自定义背景高亮颜色
- **高亮透明度**：调整背景高亮的透明度（0-100%）
- **主颜色**：自定义标记的主颜色，会影响图标和文本样式的颜色

#### 2.2 标记时间配置
- **标记有效时长**：设置单个链接标记的有效时间（1小时、1天、7天、30天、永久）
- **时间显示格式**：选择标记时间的显示格式
- **自定义时间格式**：当选择「自定义」格式时，可以手动输入时间格式
- **显示标记时间**：控制是否在标记链接旁显示标记时间

#### 2.3 自动化规则
- **启用自动化规则**：控制黑白名单是否生效
- **黑名单**：添加不希望标记链接的域名
- **白名单**：添加仅希望标记链接的域名
- **支持模糊匹配**：例如 `*.baidu.com` 可以匹配所有百度子域名

#### 2.4 性能与数量配置
- **单网页标记数量上限**：设置单个网页最多可以标记的链接数量

### 3. 历史记录页面 (History)
在弹窗页面点击「历史记录」按钮，进入标记历史管理页面，包含以下功能：

- **导出数据**：将所有标记数据和配置导出为 JSON 文件
- **导入数据**：从 JSON 文件导入标记数据，支持覆盖或追加模式
- **清除过期记录**：批量清除所有已过期的标记记录
- **刷新列表**：重新加载历史记录列表
- **查看详细信息**：查看每条标记记录的详细信息（链接地址、文字、时间、域名等）
- **单独删除**：删除单条标记记录

## 三、配置项说明

### 1. 核心配置
- **enabled**：全局启用/禁用插件
- **maxLinksPerPage**：单网页标记数量上限

### 2. 样式配置
- **iconStyle**：前缀图标样式
- **iconSize**：图标大小
- **bold/italic/strikethrough**：文本样式
- **highlight**：是否启用背景高亮
- **highlightColor**：背景高亮颜色
- **highlightOpacity**：背景高亮透明度
- **mainColor**：标记主颜色

### 3. 时间配置
- **duration**：标记有效时长
- **timeFormat**：时间显示格式
- **customTimeFormat**：自定义时间格式
- **showTime**：是否显示标记时间

### 4. 规则配置
- **autoRulesEnabled**：是否启用自动化规则
- **blacklist**：黑名单域名列表
- **whitelist**：白名单域名列表

## 四、导入导出操作步骤

### 1. 导出数据
1. 进入历史记录页面
2. 点击「导出数据」按钮
3. 浏览器会自动下载一个名为 `link-marker-data-时间戳.json` 的文件
4. 该文件包含了所有标记数据和插件配置

### 2. 导入数据
1. 进入历史记录页面
2. 点击「导入数据」按钮
3. 在弹出的导入选项中，选择要导入的 JSON 文件
4. 选择导入模式：
   - **覆盖现有数据**：完全替换当前的所有数据
   - **追加到现有数据**：在现有数据基础上添加新数据
5. 点击「确认导入」按钮
6. 导入成功后，页面会自动刷新，显示导入的数据

## 五、常见问题解答

### 1. 标记不显示怎么办？
- 检查插件是否已启用
- 检查当前域名是否在黑名单中
- 检查是否达到了单网页标记数量上限
- 尝试刷新页面

### 2. 标记自动消失了怎么办？
- 检查标记的有效时长设置
- 标记可能已经过期，系统会自动清除过期的标记

### 3. 如何批量管理标记？
- 进入历史记录页面，可以查看所有标记记录
- 支持批量清除过期记录
- 支持导出/导入所有标记数据

### 4. 如何让插件只在特定网站工作？
- 进入选项设置页面
- 启用自动化规则
- 在白名单中添加希望标记的域名

### 5. 如何让插件不在特定网站工作？
- 进入选项设置页面
- 在黑名单中添加不希望标记的域名

## 六、技术说明

### 1. 数据存储
- 所有标记数据和插件配置都存储在 Chrome 的本地存储中（chrome.storage.local）
- 数据会在浏览器重启后保留
- 可以通过导出/导入功能备份和迁移数据

### 2. 性能优化
- 后台脚本每30分钟检查一次过期记录，自动清理过期数据
- 支持设置单网页标记数量上限，避免过多标记影响页面性能
- 采用事件委托和批量更新策略，减少DOM操作次数

### 3. 安全性
- 插件仅在本地运行，不会向任何服务器发送数据
- 所有数据都存储在本地，保护用户隐私
- 权限申请最小化，仅申请必要的权限
